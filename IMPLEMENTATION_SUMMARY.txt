================================================================================
API ENDPOINTS (11 Total)
================================================================================

SYSTEM:
  GET /health                    - Health check

CHAT MANAGEMENT:
  POST /api/chat/create          - Start new conversation
  POST /api/chat/{chat_id}/message    - Send message to agent
  GET /api/chat/{chat_id}/history     - Get conversation history
  GET /api/chat/{chat_id}/clear       - Clear conversation history
  GET /api/chats                 - List active chat sessions

KNOWLEDGE BASE:
  POST /api/kb/initialize        - Load PDFs into vector database
  GET /api/kb/info               - Get KB statistics
  POST /api/kb/search            - Direct KB search

TICKETS:
  GET /api/tickets               - List all tickets
  GET /api/tickets/{ticket_id}   - Get specific ticket

================================================================================
MESSAGE PROCESSING FLOW
================================================================================

User Input: "I can't log in to my account"
        ↓
1. Search Knowledge Base
   - Generate embedding for query
   - Find top 5 similar documents
   - Results: FAQ (92%), Troubleshooting (88%)
        ↓
2. Get Ticket Context (if provided)
   - Retrieve ticket from system of record
   - Add to context
        ↓
3. Build LLM Prompt
   - System message (professional support agent)
   - Ticket context (if available)
   - Knowledge base results (with sources)
   - Conversation history
   - User message
        ↓
4. Call ChatGPT API
   - Process with full context
   - Generate contextual response
        ↓
5. Format Response
   - Agent response text
   - Citation to KB sources
   - Ticket information
   - Similarity scores
        ↓
Response: "I see you're having login issues. Based on our FAQ..."
